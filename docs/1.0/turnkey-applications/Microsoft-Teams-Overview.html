Introduction

Telestax MS Teams helps Communications Providers enable their Business Customers to send and receive SMS using Microsoft Teams and to manage and monitor their usage. Business Customers can send two-way SMS.

To enable Business Customers on MS Teams, below steps need to be followed. * Request Access Tokens from Business Customers MS Teams Admin as explained in Doc here * Create Business Customer and setup tokens received from previous step * Create a mapping (via the API or the user interface of the Console) within the Business Customer * Start sending messages

Prerequisite

In order to complete the aforementioned steps, you will need:

  • Microsoft Teams setup for your Enterprise Customers.

  • Must have followed the steps mentioned in the document here

  • Working knowledge of cURL and a way to make HTTP API calls (either using cURL or Postman. We recommend Postman if you are not familiar with cURL) Or you can directly use the UI Console exposed by Restcomm.

  • An understanding of Restcomm API for Microsoft Teams or Console on how to add mappings.

  • An activated Restcomm Cloud account.

You need to reach out to your Technical Account Manager to make sure your Account has been correctly set up for Microsoft Teams SMS enablement.

Terminology

As part of this documentation, the following terms will be used to describe specific individuals, concepts, and organizations.

Communications Provider: Communications Provider generally refers to any Telestax Partner providing communications products, applications, capabilities, or services. The term can refer to operators, carriers, providers, suppliers, or any other entity that delivers communications capabilities to enterprise or business customers.

Business Customer: An enterprise or business customer that relies on a Communications Provider to deliver communications products, capabilities, or services to customers or end users.

Setting up your First Mapping

You must have a valid, working Restcomm account in order to complete this step. If you do not have one, you can contact us.

The purpose of Microsoft Teams mapping is to associate a phone phone number with an existing Microsoft Teams Id. For example, imagine you have a user with the Microsoft Teams id of mac@myserver.com and Restcomm number is +1-919-555-1212.

Mapping will tell the Restcomm service for Microsoft Teams to send all inbound SMS messages sent to phone number (+1-919-555-1212) to Microsoft Teams Space where mac@myserver.com is a part of. Additionally, any time Joe sends an outbound SMS from Microsoft Teams account, it will be sent from a mapped phone number.

You can set up your mapping in two different ways, either via the Restcomm Message Exchange Console for Microsoft Teams or via the API.

Setting up a new mapping via Console

  • Log in to Restcomm Microsoft Teans Console via your Restcomm cloud account.

  • Create new Mapping as explained in Section-4.2 Add Mappings of Microsoft Teams console Docs

  • Before creating mapping, you need to create Enterprise. This is a very important step as Microsoft Teams Bot has one to one mapping with Enterprise.

  • Add Bot Details(for each enterprise) as explained in section 2.3 Manage Bot details in Docs :Microsoft Teams console

Receiving an SMS/message in Microsoft teams

Once you have completed all of the steps above, you are ready to test. Send a message from PSTN number to mapped number. You should see the message appear in Microsoft Teams Space.The message should appear in new Space if you are receiving a message for the first time. All subsequent messages appear in the same space. Space name by default is Phone Number: xxxxx where the phone number is from which the message was initiated (sender) and xxx is the name provided while creating the mapping. You can rename space to something more meaningful, for example, Ryan’s Mobile Phone

Replying to received SMS/message

To reply to received message follow below steps

  • In newly created space, refer to Bot with @ prefix and then type your message

  • Hit enter and your message will be sent as SMS to the mobile number of the receiver.

Sending an SMS/message to new Number from Microsoft Teams

To send SMS to new number, follow below steps

  • Start a conversation with the newly created Bot. You will find the Bot in the App’s section and when you click on the Bot’s name, you need to click on Open. This is a one time process.

  • Send any private message to this Bot

  • It should respond back with Card that allows you to enter the receiving party’s phone number

  • Enter the receiving party’s phone number and hit Submit. It should create a new Space with "receiver Phone number:Mapping Name" as Space name

  • In newly created space, refer to Bot with "@" prefix and then type your message

  • Hit Enter and your message will be sent as SMS to the mobile number of the receiver.
